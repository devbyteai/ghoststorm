Understanding the Attack Surface

Understanding how attackers create thousands of fake accounts helps build better defenses.

Phone Verification Bypass

SMS Services
  • SMSPool, 5sim, TextVerified
  • $0.10-0.50 per verification
  • Numbers from 100+ countries
  • API for automation
SIM Farms
  • Physical SIM card racks
  • Automated SMS forwarding
  • Real carrier numbers
  • Harder to detect
Detection: Track phone number age, carrier type, geographic consistency with IP

Device Fingerprint Spoofing

Emulators
  • Android Studio AVD
  • Genymotion
  • BlueStacks (modified)
Spoofed Values
  • IMEI / MEID
  • Android ID
  • Advertising ID
  • Build.* properties
App Cloners
  • Parallel Space
  • Dual Space
  • Multiple accounts/device
Detection: Check for emulator artifacts, root/jailbreak, app tampering signatures

IP / Network Diversity

Mobile Proxies (Best)
  • 4G/5G carrier IPs
  • Shared by real users
  • Nearly impossible to block
  • $3-10/GB
Residential Proxies
  • Home ISP IPs
  • Good reputation
  • Some detection possible
  • $1-5/GB
Datacenter (Worst)
  • Cloud provider IPs
  • Easy to detect ASN
  • Often pre-blocked
  • $0.10/IP

Behavior Evasion Patterns

Warming New accounts browse passively for 1-7 days before engaging. Mimics real user onboarding.
Gradual Start with 1-2 follows/day, gradually increase. Avoid sudden spikes.
Randomized Random delays (30s-5min), varied session lengths, different times of day.
Content Post some content, have a bio, profile picture. Empty profiles are suspicious.

Signals that indicate an account may be fake or automated.

Signal Description Risk
Immediate engagement New account follows/likes within minutes of creation High
No profile data Missing bio, profile picture, or posts Medium
Generic username Pattern like user123456789 or random characters Medium
Engagement-only Follows/likes but never posts own content High
Burst activity High activity in short bursts, then silence High
Device fingerprint reuse Same device ID across multiple accounts High
Datacenter IP IP belongs to AWS, GCP, or known proxy provider High
Coordinated timing Multiple accounts act within same second/minute High
Follow/unfollow pattern Follows then unfollows within hours (F4F abuse) Medium
Geographic mismatch IP location doesn't match phone number country Medium

Behavioral Analysis

Mouse/Touch Patterns
  • Movement velocity curves
  • Click precision distribution
  • Scroll acceleration patterns
  • Touch pressure (mobile)
Session Patterns
  • Time between actions
  • Session duration distribution
  • Page visit sequences
  • Scroll depth patterns

Network Analysis

IP Reputation Score Check against abuse databases (Spamhaus, AbuseIPDB)
ASN Classification Datacenter vs Residential vs Mobile carrier
VPN/Proxy Detection Known VPN IP ranges, proxy headers
Geographic Consistency IP location vs timezone vs language settings

Account Graph Analysis

Cluster Detection: Accounts that only follow/engage with each other form detectable clusters in the social graph.

Engagement Patterns: Real users have diverse engagement; bots often target the same accounts.

Creation Time Correlation: Accounts created within same time window with similar patterns.

Shared Resources: Same IP, device, or phone number across multiple accounts.

How platforms fight back against fake accounts and automation.

Shadow Banning

Account appears normal to owner, but content is hidden from others. Engagement doesn't count. User doesn't know they're banned.

Rate Limiting

Slow down suspicious accounts. Add delays, reduce API limits, show more CAPTCHAs. Makes automation expensive/slow.

Re-verification

Force suspicious accounts to re-verify phone/email. Triggered by IP change, suspicious activity, or periodic checks.

Mass Purges

Periodic ban waves using ML models. Train on known bots, detect similar patterns, mass-ban clusters of fake accounts.
